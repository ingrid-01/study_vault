# 01-1 웹 프로그래밍이란? #

## Programming? ##
- 프로그램 만드는 과정 = 프로그래밍
	- computer언어로 명령어를 나열하는 행위
- 코딩 = Source Code를 작성하는 행위
- 프로그래밍 = 문제 분석, 논리적 해결방안 찾기 + 코딩

## Web Programming? ##
- web browser와 관련된 프로그래밍

- 웹사이트: server cpu + client cpu간에 정보 주고 받기 = 사용자에게 보여주기
	- 사용자의 검색 요청 - 처리는 서버, 웹 브라우저 보여주는 클라이언트
- 백엔드 → 서버에서 사용자의 요청을 처리 + 데이터 관리 (PHP, DB, ASP, JAVA etc.)
- 프론트엔드 → 서버에서 받아온 정보를 보여주는 (HTML, CSS, JS)
# 01-2 JS로 무엇을 할 수 있을까? #

JS → 원래 용도 = WS에 움직이는 효과만을 주는 언어 = 필수 요소 X
- BUT 지금은 JS로 서버 만들기 가능, 웹에서 작동하는 프로그램 만들기 가능 등
## 1. WS 동적으로 만들기 ##
- HTML + CSS로 외관 만들고 + JS로 동적으로 만든다
	- e.g. 하위 메뉴를 펼치기, 영화 포스터 < 와 >로 좌우로 넘겨 보기 
## 2. WB에서 실행되는 프로그램 만들기 ##
- e.g. 버스 정보 검색 프로그램
	- 버스 노선 + 정류장 정보 저장 기능
	- 지도에 현재 장소 표시 기능 = 다 JS로 구현 가능
## 3. 서버를 구성하고 서버용 프로그램 만들기 ##
- 과거에 서버 프로그램 만들기 = PHP, ASP.NET, Java 등
- BUT 지금은 JS로만 가능
	- e.g. Node.js (JS 프레임워크)
		- (프레임워크 = 기본으로 제공하는 소스를 수정하거나 추가하는 방법, 일종의 틀)

# 01-3 JS의 특징은 무엇일까? #

## 모든 WB에서 작동한다 ##
- JS = WB를 위한 언어
- 대부분 WB에서 작동함 
	- BUT JS최신 문법을 즉시 사용할 수 있는 건 X

- JS versions
	- JS는 ECMAScript라는 이름으로 바뀌고 1년에 한 번씩 기능이 업그레이드 중
		- 2015년 발표 6번째 판 (edition) - ECMAScript 2015 (ES6) 부터 ECMAScript 2025 (ES16)

- JS는 WB만 있으면 어디에서나 실행 가능 = 별도의 프로그램 필요 X
## 풀스택 웹 개발뿐 아니라 다양한 용도의 프로그램 만들기 ##
- JS는 클라이언트에서만 사용 X
- FE & BE를 모두 제어 가능
- JS로만으로 App 개발 가능
- 아두이노, 드론 (사물인터넷 IoT: Internet of Things)환경에서 실행되는 프로그램 개발 가능
	- e.g. 로보틱스 프레임워크 (cylonjs.com)
## 다양한 JS 공개 API를 사용 가능 ##
- WS → 버스 or 날씨 정보 필요 → 날씨 정보 프로그램 → 전국의 날씨 정보 DB필요 → 수집? 
	- API 사용
		- API = 어떤 데이터를 다른 사람이 가져갈 수 있도록 미리 준비한 체계
			- e.g. 기상청에서 만든 API
			- FB, YouTube, Naver, 관공서 등 API 공개 중
		- THUS, JS만 알아도 데이터 활용하여 웹 프로그램 만들기 가능!
## 다양한 라이브러리와 프레임워크를 사용 가능 ##
- JS Library = JS를 미리 구현해 놓은 기능을 묶어 놓은 것
	- e.g. jQuery JS Library = WS를 동적으로 개발하기 위해 필요한 기능을 짧은 코드로 사용할 수 있게 만든 Library
	- I'm in charge of building the whole things

- JS Framework = Framework에서 기본으로 제공하는 소스를 수정하거나 추가하는 방법으로 웹 프로그램을 만들 수 있게 미리 준비한 틀
	- e.g. React, Angular, Vue
	- a guideline is given to follow to build the thing
