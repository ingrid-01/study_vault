- 프로그래밍 = 커뮤니케이션
	- 무엇을 실행하고 싶은지 정의해야 함
		1. 해결해야 할 문제(요구사항) 명확한 이해 필요
		2. 적절한 문제 해결 방안 정의 필요

	- ^ 문제 해결 능력 필요
		- 문제 - 분해(decomposition) + 자료 정리하고 구분(modeling) 순서에 맞게 행위를 배열

	- 프로그래밍 = 정확 + 상세 요구사항을 설명 = 코드
		- Computational thinking

- 약속된 구문 = syntax
- 의미 = semantics 

^ 이 둘의 조합으로 표현된다

## 1.3 구문과 의미
Noam Chomsky - 언어의 의미는 문맥에 있는 것이지, 문법에 있는 것이 아니다

"Colorless green ideas sleep furiously." → 문법적 문제는 없지만, 의미는 없다. 프로그래밍도 마찬가지다. 

```js
const number = 'string';
console.log(number * number); // NaN
```
- 작성된 코드는 해결 방안의 구체적 구현물이다
- **요구사항이 실현 (문제가 해결)** 되어야 의미가 있다
