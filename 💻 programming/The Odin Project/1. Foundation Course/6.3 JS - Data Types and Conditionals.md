## BigInt ##
In JS, "number" has to be btw `2^53 - 1` and `-(2^53 - 1)`
- "number" can store integers up to `1.7976931348623157 * 10^308`
- BUT outside of ±(2^53 - 1) = precision error; approx. value may be stored
```js
console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992
```

- Most of the time, that's enough to run JS
	- BUT in cryptography/microsecond-precision timestamps = bigint needed

`BigInt` type = integers of arbitrary length
	created by appending `n` to the end of an integer
```js
// the "n" at the end means it's a BigInt
const bigInt = 1234567890123456789012345678901234567890n;
```

## String ##
string = surrounded by quotes; piece of text; FUNDAMENTAL building block of the language
```JS
let str = "Hello";
let str2 = 'Single quotes are ok too';
let phrase = `can embed another ${str}`;
```

- 3 types of quotes in JS
	1. Double quotes: "Hello".
	2. Single quotes: 'Hello'.
	3. Backticks: `Hello`.

- Double, single quotes = simple quotes; no difference
- Backticks = extended functionality quotes; "template literal"; embed variables & expressions into a string by wrapping them in ${...}
	- can declare template literals over multiple lines
```js
let name = "John";

// embed a variable
alert( `Hello, ${name}!` ); // Hello, Johnn!

// embed an expression
alert( `the result is ${1 +2}` ); // the result is 3
```
↳ expression in `${...}` = evaluated & result becomes part of string
```js
alert( "the result is ${1 + 2}" ); // the result is ${1 + 2}; no effect in embedding functionality
```

- joining 2 variables
```js
const one = "Hello, ";
const two = "how are you?";
const joined = `${one}${two}`;
console.log(joined); // "Hello, how are you?"
```
↳ joining strings together = "Concatenation"

##### JS String slice() #####

## 1. Comparisons ##

- **Greater/less than**: `a > b, a < b`
- **Greater/less than or equals**: `a >= b, a <= b`
- **Equals**: `a ==b`
	- double equality sign means the equality test (loose equality)
		- checks if 2 values are the same BUT allows type conversion (JS tries to make them the same type before comparing)
```js
5 == "5"; // true (string "5" is converted to number 5)
```
- single equality sign means an assignment
		- "put the value on the right into the variable on the left" = it stores value
```js
let x = 5; // put 5 inside the box called x
```

- triple equals 
	- strict equality test
	- checks if 2 values are the same w/o type conversion
	- both value & type MUST match
```js
5 === "5"; // false (number is not the same type as string)
5 === 5; // true
```

∴
- single equals = "put this in the box"
- double equals = "Are these the same? don't mind if they look a bit different"
- triple equals = "Are these exactly the same?"

- **Not Equals**: ≠ in JS is `a != b`
### Boolean is the results ###
ALL comparison operators return a boolean value:
- `true` = yes, correct, the truth
- `false` = no, wrong, not the truth

```js
alert( 2 > 1); // true (correct)
alert( 2 == 1 ); // false (wrong)
alert( 2 != 1); // true (correct)
```
So a comparison result can be assigned to a variable
```js
let result = 5 > 4; // assign the result of the comparison
alert( result ); // true
```
### String comparison ###
- to compare string = JS uses "dictionary" or "lexicographical" order
	- Strings are compared letter-by-letter
	- Lowercase `a` is greater than capital letter `A` = lowercase char has greater index in the internal encoding table JS uses (Unicode)
```js
alert( 'Z' > 'A' ); // true
alert( 'Glow' > 'Glee' ); // true
alert( 'Bee' > 'Be' ); // true
```

- To compare 2 strings:
	1. Compare the first character of both strings
	2. If the 1st char from the 1st string is GREATER (or LESS) than the other string's, then the 1st string is GREATER (or LESS) than the second. We're done. 
	3. Otherwise, if both strings' 1st character are the same, compare the 2nd characters the same way
	4. Repeat until the end of either string.
	5. If both strings end at the same length, then they are equal. Otherwise, the LONGER string is GREATER.

- 1st example above: `Z > A` gets to the result at the 1st step
- 2nd example: `Glow and Glee` needs more steps as strings are compared char-by-char
	- G is same as G
	- L is the same as L
	- o is greater than e. Stop here. The first string is greater.

## 3. Logical Operators ##
- 4 logical operators in JS 
	- || (OR)
	- && (AND)
	- ! (NOT)
	- ?? (Nullish Coalescing) 
- While called "logical", these operators work on any data type (not just boolean) and may return non-boolean values
### || (OR) ###
```js
result = a || b;
```
- With boolean logic, `a || b` is `true` if **either** is `true`; only `false || false` yields `false`
```js
alert( true || true ); // true
alert( false || true ); // true
alert( true || false ); // true
alert( false || false ); // false
```
- if operands ≠ booleans → converted to a boolean for the evaluation
	- number 1  = `true`
	- number 0 = `false`
```js
if(1 || 0) { // works just like if( true || false)
	alert( 'truthy!' );
}
```
- OR | | used in a `if` statement to test if ANY of the given condition is `true`
```js
let hour = 9;

if (hour < 10 || hour > 18) {
	alert( 'The office is closed.' );
}
```
### OR "||" finds the first truthy value ###
```js
result = value1 || value2 || value3;
```

**ROLE of OR (||)**
- evaluates operands left → right
- each operand converted to boolean
	- if `true` = stops & returns the OG value of that operand
	- if operands have been evaluated (i.e. were `false`), returns the last operand

- a value is returned in its OG form, w/o the conversion
	- chain of OR || returns the first truthy value or the last one if no truthy value is found
```js
alert( 1 || 0); // 1 (1 is truthy)

alert( null || 1 ); // 1 (1 is the first truthy value)
alert( null || 0 || 1 ); // 1 (the first truthy value)

alert( undefined || null || 0 ); // 0 (all falsy, returns the last value)
```
↑ This leads to:

1. Getting the first truthy value from a list of variables or expressions 
	- i.e. `firstName`, `lastName` and `nickName` variables, all optional (i.e. can be undefined or have falsy values
```js
let firstName = "";
let lastName = "";
let nickName = "SuperCoder";

alert( firstName || lastName || nickName || "Anonymous"); // SuperCoder
```
↳ If all variables were falsy, "Anonymous" would show up

2. Short-circuit Evaluation
	- Another feature of OR || operator = "short-circuit" evaluation
		- || processes arguments until the first truthy value is reached + value is returned immediately w/o touching the other argument
		- IMPORTANT when operand is an expression w a side effect - e.g. variable assignment or a function call
```js
true || alert("not printed");
false || alert("printed");
// only the second messageis printed
// 1st line the OR || operator stops the evaluation immediately upons seeing `true` = `alert` isn't run
```
#### Summary ####
- `a || b` returns the first truthy value it finds, or the last value if all are falsy
- it doesn't just give `true/false`; returns one of the actual operands
##### Rules #####
1. `true || anything` → always true
2. `false || value` → returns `value`
3. It stops as soon as it finds a truthy value (short-circuiting)
```js
console.log( true || false ); // true
console.log( false || "Hello"); // "Hello"
console.log( 0 || null || "Hi"); // "Hi"
console.log( "" || 0 || false ); // false (all falsy, so return last one)
```
##### Main Use #####
- provide defaults for any falsy value (0, "", false, null, undefined, NaN)
```js
let user = "";
let name = user || "Guest";
console.log(name); // "Guest"
```
### && (AND) ###
```js
result = a && b;
```

- Boolean logic: only `true && true` yields `true`; otherwise it's `false`
- In JS:
	- evaluates left to right
	- converts each to boolean:
		- returns the first falsy value it sees
		- if all are truthy, returns the last operand
- Example:
	- `1 && 0` → `0` (first falsy)
	- `1 && 5` → `5` (all truthy, return last)
```js
alert( true && true ); // true
alert( false && true ); // false
alert( true && false ); // false
alert( false && false ) // false
```

An example w `if`:
```js
let hour = 12;
let minute = 30;

if (hour == 12 && minute == 30) {
	alert( 'The time is 12:30');
}
```
- Just as w OR, any value is allowed as an operand of AND:
```js
if (1 && 0) { // evaulated as true && false
	alert( "won't work, because the result is falsy" );
}
```
### AND "&&" finds the first falsy value ###
```js
results = value1 && value2 && value3;
```

&& operator:
- Evaluates operands left to right
- For each operand, converts it to a boolean (true or false)
	- if `false` = stops & returns the OG value of that operand
- All evaluated operands (all truthy), returns last operand
∴ AND returns 1) first falsy value OR 2) last value if none were found
- Rules similar to OR, 
	- only difference:
		- AND returns the first **Falsy** 
		- OR returns first **Truthy**

```js
// if the 1st operand is truthy, 
//AND returns the second operand:
alert( 1 && 0 ); // 0
alert( 1 && 5 ); // 5

// if the first operand is falsy, 
// AND returns it. The second operand is ignored
alert( null && 5 ); // null
alert( 0 && "no matter what" ); // 0
```

```js
alert( 1 && 2 && null && 3 ); // null
alert( 1 && 2 && 3 ); // 3, the last one
```
### 4. Operator Precedence ###
- `&&` ahs higher precedence than `||`
- Example:
```js 
a && b || c & d
// interpreted as:
(a && b) || (c && d)
```
- Always use parentheses if you're unsure about grouping
### 5. `!` (NOT) Operator
- Syntax:
```js
result = !value;
```
- Behaviour:
1. Converts `value` to boolean
2. Returns its inverse (`true` → `false`, `false` → `true`)
3. Double NOT(`!!`) is a quick way to convert any value to a boolean:
```js
!!value // true if truthy, false if falsy
```
- `!` has the highest precendece among logical operators (executes before `&&` or `||`)

## Key Points from the Page

### 1. **Logical Operators in JavaScript**

- JavaScript has four logical operators: `||` _(OR)_, `&&` _(AND)_, `!` _(NOT)_, and `??` 
    
- While called “logical,” these operators work on any data type (not just boolean) and may return non-boolean values.
    
---

### 2. **`||` (OR) Operator**

- Syntax:
    
    `result = a || b;`
    
- With boolean logic, `a || b` is `true` if **either** is `true`; only `false || false` yields `false` [JavaScript.info](https://javascript.info/logical-operators).
    
- In JavaScript’s flexible behavior:
    
    - Evaluates left to right.
    - Converts each operand to boolean:
        - If an operand is _truthy_, it **returns that original value** (not `true`).
        - If all are _falsy_, returns the **last operand** 
            
- Common patterns:
    
    - `value1 || fallback || defaultValue` → picks the first truthy.
    - Great for short-circuiting and default assignments 
        
- **Short-circuit evaluation**: stops as soon as a truthy is found, skipping remaining expressions
    

---

### 3. **`&&` (AND) Operator**

- Syntax:
    
    `result = a && b;`
    
- Boolean logic: only `true && true` yields `true`; otherwise it's `false`
    
- In JavaScript:
    
    - Evaluates left to right.
    - Converts each to boolean:
        - Returns the **first falsy** value it sees.
        - If all are truthy, returns the **last operand** 
            
- Example:
    
    - `1 && 0` → `0` (first falsy).
    - `1 && 5` → `5` (all truthy, return last)
---

### 4. **Operator Precedence**

- `&&` has higher precedence than `||`.
    
- Example:
    
    `a && b || c && d // interpreted as: (a && b) || (c && d)`
    
- Always use parentheses if you're unsure about grouping

---

### 5. **`!` (NOT) Operator**

- Syntax:
    
    `result = !value;`
    
- Behavior:
    
    1. Converts `value` to boolean.
    2. Returns its **inverse** (`true` → `false`, `false` → `true`)
        
- Double NOT (`!!`) is a quick way to convert any value to a boolean:
    
    `!!value  // true if truthy, false if falsy.`
    
- `!` has the **highest precedence** among logical operators (executes before `&&` or `||`)

`if` condition to check that `age` is NOT btw `14 & 90` inclusively:
```js
if (!(age >= 14 && age <= 90))

if (age < 14 || age > 90)
```
- `!(A && B)` is logically the same as `!A || !B` (De Morgan’s law).
- Variant 1: Wrap the “inside range” condition in `!`.
- Variant 2: Write the “outside range” directly with `<` and `>`.

## Study Notes (for your own reference)

- **`||`**: returns the **first truthy** value (or the last if none).
    
- **`&&`**: returns the **first falsy** value (or the last if all are truthy).
    
- Use `||` for **fallback values** (e.g., `userName || "Guest"`).
    
- Use `&&` for **conditional execution** or chain success (e.g., `user && user.name`).
    
- **Operator Precedence**: `!` → `&&` → `||`.
    
- **Double NOT (`!!`)**: quick way to convert any value to a `true/false`.
    
- **Avoid abusing** `||` or `&&` in place of `if`—prefer clarity with `if` for readability unless the pattern is idiomatic.

# Conditionals - MDN #
## if...else statements ##
```js
if (condition) {
	/* code to run if condition is true */
} else {
	/* run some other code instead */
}
```
1. keyword `if` followed by parentheses
2. A condition to test (placed inside parentheses) - typically "is this value bigger than this other value?" OR "does this value exist?"
	- condition makes use of the comparison operators and returns `true or false`
3. A set of curly braces (some code in it) - can be any code + only runs if the condition returns `true`
4. The keyword `else`
5. Another set of curly braces (more code) - can be any code + only runs if the condition is `false`

∴ IF the CONDITION returns `true` = run code A
ELSE, run code B

Also, `else` is not always required. It runs without it.
```js
if (condition) {
	/* code to run if condition is true */
}

/* run some other code */
```
BUT this means the second code ALWAYS runs, no conditional statement

- sometimes `if...else` statement are written w/o curly braces
	- NOT RECOMMENDED
	- hard readability

## else if ##
- when we want MORE THAN 2 outcomes
	- `else if` 
		- each extra choice requires an additional block to put in btw `if () {}` and `else {}`
```html

	<option value=""--Make a choice--</option>

	<option value="rainy">Rainy</option>

```
```js
const select = document.querySelector("select");
const para = document.querySelector("p");

select.addEventListener("change", setWeather);

function setWeather() {
	const choice = select.value;

	if (choice === "sunny") {
		para.textContent =
			"It is nice and sunny outisde today. Wear shorts! Go to the beach, or the park, and get an ice cream.";
	} else if (choice === "rainy") {
	para.textContent =
		"Rain is falling outside; take a rain coat and an umbrella, and don't stay out for too long.";
	} else if (choice === "snowing") {
	para.textContent =
		"The snow is coming down - it is freezing! Best to stay in with a cup of hot chocolate, or go build a snowman.";
	} else if (choice === "overcast") {
	para.textContent = 
		"It isn't raining, but the sky is grey and gloomy; it could turn any minute, so take a rain coat just in case.";
	} else {
	para.textContent = "";
	}
}
```
1. HTML `<select>` element = can make diff. weather choices + a simple paragraph
2. JS = store a reference to both `<select>` and `<p>` + adds an event listener to `<select>` elements so when value is changed, `setWeather()` function is run
3. when this function is run: 
	- first set variable: `choice` to `<select>` element
	- then use conditional statement to show diff. text inside the paragraph depending on the `choice` value
	- 1 = `if` block
	- Rest = `else if` block
4. Very lat choice `else` block = last resort option; code inside will run if non of the condition are true. 
	- this case, it serves to empty the text out of the paragraph if NOTHING is selected
