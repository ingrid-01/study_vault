# Basic Syntax #
`div.bold-text {`
	`font-weight: 700;`
`}`

- `div.bold-text` = selector
- `font-weight` = property
- `700` = value

CSS = various rules
Rule = selector + semicolon-separated list of declarations (property-value pair)
# Selectors #
- what is "selected" for each rule
## Universal Selector ##
- select every type of element
	- syntax = `*` 
E.g. 
```css
* {
   color: purple;
}
```
↳ this means that every element would have `color:purple` style applied to it
## Type Selector ##
- a.k.a. element selector → all of a selected element is applied
E.g.
```html
<!-- index.html --> 

<div>Hello, World!</div>
<div>Hello again!</div>
<p>Hi...</p>
<div>Okay, bye.</div>
```
```css
/*styles.css*/

div {
	color: white;
}
```
↳ all 3 `<div>` will have `color:white` applied to them
## Class Selectors ##
(Attribute selector) - selects all elements w given class
```html
<!-- index.html -->

<div class="alert-text">Please agree to our terms of service.</div>
```
```css
/* styles.css */

.alert-text {
  color: red;
}
```
↳ note the `.` before `alert-text` (also, case-sensitive)

also note - if class name begins w a number ≠ match

ALSO NOTE - class atts. can be listed in a single element → e.g. `class="alert-text severe-alert"`
	↳ whitespace = separate class; ∴ multi-worded name ALWAYS hyphend
## ID Selectors ##
similar to Class Selectors
but instead of "class", it matches "id"
* can't be repeated on a single page + can't contain whitespace
```html
<!-- index.html -->

<div id="title">My Awesome 90's Page</div>
```
```css
/* styles.css */

#title {
  background-color: red;
}
```
↳ note the `#` before the selector + case-sensitive

same as class selector, can't start with a number
## The Grouping Selector ##
```css
.read {
  color: white;
  background-color: black;
  /* several unique declarations */
}

.unread {
  color: white;
  background-color: black;
  /* several unique declarations */
}
```
↳ 2 groups of elements, share style declarations
	- `color: white;` and `background-color: black;` are shared
	- this can be grouped!
```css
.read,
.unread {
  color: white;
  background-color: black;
}

.read {
  /* several unique declarations */
}

.unread {
  /* several unique declarations */
}
```
## Chaining Selectors ##
```html
<div>
  <div class="subsection header">Latest Posts</div>
  <p class="subsection preview">This is where a preview for a post might go.</p>
</div>
```
if we want to sort both `subsection` + with the class `header`, chain them ↓
```css
.subsection.header {
	color: red;
}
```
↳ NOTE: no space btw the 2 selectors + `.` before each selectors
- works for class and id selectors
- doesn't work for type selectors

For class & id selectors:
```html
<div>
  <div class="subsection header">Latest Posts</div>
  <p class="subsection" id="preview">
    This is where a preview for a post might go.
  </p>
</div>
```
```css
.subsection.header {
  color: red;
}

.subsection#preview {
  color: blue;
}
```
## Descendant Combinator ##
There are 4 types of combinator. 
Most commonly used one = descendant combinator
```html
<!-- index.html -->

<div class="ancestor">
  <!-- A -->
  <div class="contents">
    <!-- B -->
    <div class="contents"><!-- C --></div>
  </div>
</div>

<div class="contents"><!-- D --></div>
```
```css
/* styles.css */

.ancestor .contents {
  /* some declarations */
}
```
↳ so, only element with class `contents` will be selected, if it's parent class is `ancestor`
↳ above example, only B and C will be selected, not D
- No limit on # of combinators

- NOTE: `space` btw the two classes
==no space btw the selector = chaining selector / 2 separate selectors chosen ==

==space btw the selector = descendant combinator==
# Properties Basics #
## Colour and Background-colour ##
`color` = text colour
`background-color` = bckgrnd colour
- both accepts actual colour names → e.g. `red` or `transparent`
- both accepts HEX, RGB, HSL values
```css
p {
  /* hex example: */
  color: #1100ff;
}

p {
  /* rgb example: */
  color: rgb(100, 0, 127);
}

p {
  /* hsl example: */
  color: hsl(15, 82%, 56%);
}
```
## Typography basics & Text-align ##
`font-family` = single value OR comma-separated list of values
- font family name → e.g. `"Times New Roman"`
	- NOTE - uses quotes
- generic family name → e.g. `serif`
	- NOTE - doesn't use quotes
	- 5 generic font family names: `serif, sans-serif, monospace, cursive, fantasy`
		- on computer screen, sans-serif = easiest to read
- When browser doesn't support first font in the list, it will use next one
	- ∴ ALWAYS use a list of font values
	- e.g. `font-family: "Times New Roman", serif;`

`font-size` = e.g. `font-size: 22px`
- NOTE - no space btw `22` and `px`

`font-weight` = boldness of text
- value can be keyword, e.g. `font-weight: bold`
	- 4 keywords: `normal (400)`, `bold (700)`, `lighter (100)`, `bolder`
- value can be number btw 1 - 1000, e.g. `font-weight: 700` 
	- usually btw 100 to 900 → BUT depends on font

`text-align` = e.g. `text-align: center`
- keywords: `center, left, right, justify`
## Image Height & Width ##
- default values = same as actual img size 
- when wanting to keep proportion, use `auto`
```css
img {
	height: auto;
	width: 500px;
}
```
- even when not adjusting, best to include these properties
	- it reserves space for img on the page
	- hence, won't drastically shift the page content + format while loading
## Adding CSS to HTML ##
### 1. External CSS ###
- most common method
- separate CSS file → link to HTML (inside `<head>` tag + void `link` element)
```html
<!-- index.html -->

<head>
  <link rel="stylesheet" href="styles.css">
</head>
```
```css
/* styles.css */

div {
  color: white;
  background-color: black;
}

p {
  color: red;
}
```
void `<link>` + `rel` att. (showing relationship btw HTML & CSS) +`href` att. + CSS file URL (commonly used: `style` or `styles.css`)

- separate HTML & CSS = smaller HTML file + cleaner look
- CSS in one place = easier to edit in a multi-page WS
### 2. Internal CSS ###
a.k.a. embedded CSS → `<style>` tag placed in `<head>` tags = `<link>` element not needed
```html
<head>
  <style>
    div {
      color: white;
      background-color: black;
    }

    p {
      color: red;
    }
  </style>
</head>
<body>
  ...
</body>
```
- useful for adding unique style to a SINGLE page of an WS
- but depending on # of rules & declaration → could cause bigger HTML file
### 3. Inline CSS ###
- not really recommended
```html
<body>
  <div style="color: white; background-color: black;">...</div>
</body>
```
- no selectors are used → direct `style` att. used
- could be used IF a unique style for a SINGLE element is used

But not recommended because:
- can quickly become messy
- elements with same style = copy & paste for each ind. element
	- LOTS OF REPETITION
- any inline CSS = overriden external & internal CSS
