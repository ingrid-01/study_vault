# Flexbox #
- Flexbox = to arrange items into ROWS or COLUMNS
	- which will flex (grow or shrink)
	- it enables responsive, efficient alignment, distribution & spacing of items despite each size
	- alternate to Floats 
		→ only allows horizontal position of our boxes
	- Flexbox = Complete Control! 

==Use Flexbox as much as possible==
![[flexbox_example_ 20250728_1739.png]]

```html
<div class="flex-container">
	<div class="one"></div>
	<div class="two"></div>
	<div class="three"></div>
</div>
```
```css
.flex-container {
	display: flex;
}

/*this selector selects all divs inside of .flex-container */
.flex-container div {
	background: peachpuff;
	border: 4px solid brown;
	height: 100px;
	flex: 1;
}
```
`flex: 1` = `flex-grow, flex-basis, flex-shrink`

To use Flexbox:
1. Apply `display: flex;` to parent container
2. Use properties like `justify-content`, `align-items`, `flex-direction` and `flex-wrap` to control alignment, orientation, flow
3. Children become flex items + can be individually controlled by: `flex-grow`, `flex-shrink` and `align-self`

Flexbox:
- simplifies layout tasks - e.g. centering, spacing, alignment
- adapt well to diff. screen sizes
- reduces need for complex float or positioning hacks
- improves readability & maintainability of layout code

## Alignment ##
- after creating a flex container → define HORIZONTAL Alignment (items)
	- `justify-content` + `flex-start`, `center` or `flex-end`
![[Screenshot 2025-07-28 at 19.26.26.png]]
- other values: `space-around` and `space-between`
## (Horizontal) Distribution ##
- `justify-content: space-around;` = evenly spread for the whole width
- `justify-content: space-between` = only adds extra space btw items

## Flex Container Direction ##
- container rendered HORIZONTAL or VERTICAL
	- defines direction of the MAIN AXIS
		- how flex items flow
		- which axis is the MAIN and which is the CROSS
- most mobile layouts = single column
	- so `flex-direction: column;` works!
- But most desktop layouts = horizontally stacked
	- ∴ `flex-direction` is USEFUL for responsive layouts
### Alignment Considerations ###
- Just doing `justify-content: center;` and `flex-direction: column;`, it hugs the left side of the page
- Rotating direction of a container = rotates direction of the `justify-content` property 
- Horizontal Center columns = must define `align-items: center`

## (Horizontal) Flex Item Order ##
![[flexboxitemorder_example_20250729_1329.png]]

- defining order WITHIN the container w/o affecting other surrounding items
	- Default value = 0
		- ↓ or ↑ value = moving the item left or right
- e.g. 
```css
.first-item {
	order: 1;
}

.last-item {
	order: -1;
}
```
↳ With 5 different items, this will only swap the order of the first and the fifth item → so 5, 2, 3, 4, 1
- but if this was to be done with direction, it'll be → 5, 4, 3, 2, 1
## (Vertical) Flex Item Alignment ##
- `align-self: (value)`
	- adding this to flex item OVERRIDES `align-items` value
- Values:
	- center
	- flex-start (top)
	- flex-end (bottom)
	- stretch
	- baseline
## Flexible Items ##
`flex` = defines width of ind. items in a flex container
- allows flexible widths
- e.g. `flex: 1;` = items stretch equally
- if one item is increased, they'll grow faster
	- e.g. `.footer-three { flex: 2; }`

Similar to `justify-content` (distributes extra space btw them)
BUT `flex` distributes space into EACH items
![[flexboxflex_example)20250729_1340.png]]
### Static Item Widths ###
- mix-and-match flexible boxes & fixed-width boxes
- `flex: initial` = explicit width property
	- allows combine static + flexible boxes
![[flexboxstaticbox_example_20250729_1350.png]]

SO → flex items with `flex:intial` will flex; flex items with `flex: (value, #)` will stay static, equal to each other
- Often used in website sidebars → usually fixed-width sidebars + flexible content block
## Flex Items & Auto-Margins ##
- Auto-margins → alternative to `extra <div>` 
	- "Divider" for flex items in the same container

`margin-left: auto;`
- auto-margins eats up ALL extra space in flex container
	- it doesn't distribute space equally; it moves the whole dang thing to one side
	- BUT w/o having to create another `<div>` = keeps HTML flatter

# Summary #

1. `display: flex;` = creates flex container
2. `justify-content` = define horizontal alignment of items
3. `align-items` = defines vertical alignment of items
4. `flex-direction` = when needing columns instead of rows
5. `row-reverse` or `column-reverse` = flips item order
6. `order` = customise order of ind. elements
7. `align-self` = vertically align ind. items
8. `flex` = create flexible boxes that stretch and shrink
