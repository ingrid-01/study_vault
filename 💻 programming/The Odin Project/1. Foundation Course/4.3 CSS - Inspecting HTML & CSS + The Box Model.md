# The Box Model #
- Most important in CSS: positioning & layout

Every single thing on an WP = rectangular box
```css
* {
	outline: 2px solid red;
}
```
![[Pasted image 20250721165225.png]]

- manipulating size of boxes + space btw them 
	- `padding` - ↑ space btw border of a box & content of the box
	- `border` - adds space btw margin & padding
	- `margin` - ↑ space btw border of a box & borders of adjacent boxes
![[Pasted image 20250721165620.png]]

- padding is INCLUDED inside `box-sizing` border-box sizing
	- border-box = width + height + padding size
	- this is same for the border size
		- e.g. boarder size + width + height + padding size = total box-sizing
		∴ rather than using the default (`content-box`), use `border-box`
## Additional Resources: Block and Inline Boxes ##

 `display` - display type
- `block` - display value = box breaks onto a new line
	- `width`, `height` respected
	- padding, margin, border cause other elements to be pushed away from the box
	- `width` NOT SPECIFIED = box extend in the inline direction = box fills the container width (100%)

HTML elements → `<h1>` and `<p>` uses `block` by default

`inline` - display type
- box won't break onto a new line
- `width` and `height` properties NOT APPLY
- top, bottom padding, margins, borders apply BUT won't cause other inline boxes to move away from the box
- left, right paddings, margins, borders apply and WILL cause other inline boxes to move away from the box

HTML elements → `<a>, <span>, <em>, <strong>` use `inline` as default

- `block` & `inline` = outer display types
- inner display types = elements inside the box layout
	- e.g.` display: flex;
		↳ other inner display types `grid`

![[Pasted image 20250722153425.png]]
- content box - `height and width`
- padding box - `padding`
- border box - `border`
- margin box - `margin`

A standard CSS box model
```css
.box {
  width: 350px;
  height: 150px;
  margin: 10px;
  padding: 25px;
  border: 5px solid black;
}
```
↳ the *actual* space taken by the box = `410px` wide = (350 + 25 + 25+ 5 + 5)
`210px` high (150 + 25 + 25 + 5 + 5)
![[Pasted image 20250722153657.png]]

### Alternative CSS box model ###
- any width is the width of the visible box on the page
- content box width = width - width for padding + border
	- USEFUL = no need to add up the border + padding to get real size of the box
```css
.box {
	box-sizing: border-box;
}
```
↑ this is same as ↓
```css
.box {
	width: 350px;
	height: 150px;
	margin: 10px;
	padding: 25px;
	border: 5px solid black;
}
```
![[Pasted image 20250722154812.png]]

For using alternative box model for ALL of my elements (common choice among devs), set box-sizing property on `<html>` + set ALL other elements to inherit that value:
```css
html {
	box-sizing: border-box;
}

*,
*::before,
*::after {
	box-sizing: inherit;
}
```

### Margins ###
- invisible space around the box - pushes other elements away
- can have +ve and -ve values
	- -ve values = overlap with other parts of the WP

Margin property:
	`margin` = controlling ALL margins
	`margin-top`, `margin-right`, `margin-bottom` and `margin-left` = controlling ind. margins
#### Margin Collapsing ####
- 2 +ve margins = 1 largest margin
- 2 -ve margins = 1 smallest (furthest from zero) will be used
- 1+ve + 1-ve margin = subtracted value

e.g. 
one paragraph = 50px margin bottom
one paragraph = 30 px margin top
∴50px margin btw the 2 

### Borders ###
standard box model = `width + height` of border is added
alternative box model = bigger the border, smaller the content box

Border property:
	`border` = controlling ALL borders
	`border-top`, `border-right`, `border-bottom` and `botder-left` = controlling ind. borders

`border-width`, `border-style`, `border-color` can also be used
↳ and these can be combined, such as:
- `border-top-width`
- `border-right-style`
- `border-left-color`
### Padding ###
- push content away from border
	- can't have -ve padding
	- background displayed BEHIND the padding
	
`padding` = controls ALL padding
	`padding-top`, `padding-right`, `padding-bottom` and `padding-left` = controls ind. padding
### For Inline boxes ###
`<span>` element is used!
	↳ 1. place `<span>` in HTML; 2. add `span` selector with box values

`display: inline-block` = middle line btw `inline` and `block`
	- use IF: I don't want an item to break onto a new line
	- use IF: I want to respect `width + height` 
	- use IF: avoid overlapping

- `display: inline-block;` = allow padding to be set on it


As a CSS preset, use:
```css
*,
*::before,
*;;after {
	box-sizing: border-box;
}
```

## Additional Resources: Margin ##
- margins = set using lengths, percentages, keyword `auto`, can have -ve values

### Sizing!! ###
`px` = absolute size, fixed
	e.g. `40px`
`em` = relative to the **font size** of the element parent. 
	e.g. If parent = `10px`, result = `40px`
`rem` = relative to the **root element** font size (`html`) = usually `16px`
	e.g. `4rem` = `64px` if root is `16px`

## Auto & Centering ##

value of `auto` = value of `0` OR whatever available space there is ont he side of the element
- USEFUL for horizontal centering
```css
.container {
	width: 980px;
	margin: 0 auto;
}
```
- specified width = given element
- left & right margin = `auto`
	↳ these 2 factors = centers an element horizontally
==`auto`only useful for  horizontal centering; not for top & bottom (Vertical centering)==
