## 자바스크립트 데이터 타입의 차이

### 두 가지 데이터 타입

**기본형(Primitive Type)**
- 숫자, 문자열, 불리언, null, undefined
- ES6에서 Symbol 추가

**참조형(Reference Type)**
- 객체와 그 하위 타입: 배열, 함수, 정규표현식
- ES6에서 Set, WeakSet, Map, WeakMap 추가

### 메모리 저장 방식의 차이

**기본형 데이터 할당**

변수 선언 시 메모리 공간을 확보하고, 데이터는 별도 공간에 저장한 후 그 주소를 변수에 할당합니다. 값을 변경하면 새로운 메모리 공간에 새 값을 저장하고 변수가 가리키는 주소를 직접 변경합니다.

**참조형 데이터 할당**

기본형보다 한 단계를 더 거칩니다. 객체의 프로퍼티들을 저장할 별도의 메모리 영역을 확보하고, 각 프로퍼티는 다시 기본형처럼 저장됩니다. 변수는 이 프로퍼티 영역의 주소를 가리킵니다.

### 중요한 차이점

**불변성**

기본형은 값 자체가 메모리에 하나만 존재하여 불변값입니다. 참조형은 프로퍼티 값을 변경해도 객체가 가리키는 주소는 변하지 않습니다.

**변수 복사**

기본형 복사 시 각 변수가 독립적으로 동작하지만, 참조형 복사 시 같은 객체를 가리키므로 한쪽을 수정하면 다른 쪽도 영향을 받습니다.

### 주소 저장 방식의 장점

값의 주소를 저장하는 방식은 초기 할당 시 비교 과정이 필요해 느리지만, 이후 비교 비용이 전혀 들지 않고 메모리 낭비를 최소화할 수 있습니다. 또한 동일한 값이 메모리에 하나만 존재하여 불변값을 보장합니다.

### 가비지 컬렉션

참조 카운트가 0이 된 메모리, 즉 어디에서도 참조하지 않는 데이터는 가비지 컬렉터에 의해 자동으로 수거됩니다.

이러한 차이를 이해하면 불변 객체, 실행 컨텍스트 등 자바스크립트의 중요한 개념들을 더 쉽게 이해할 수 있습니다.
