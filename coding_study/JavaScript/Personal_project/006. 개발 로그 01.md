## 📝 1. Insight Deck 개발 히스토리 (Development Log)

이 노트는 프로젝트의 시작부터 현재(v1.6)까지의 개발 과정을 요약한 것입니다.
### **Phase 1: 기본 구조 및 UI 구축 (Foundation)**

- **레이아웃:** 
	- 좌측 사이드바(통계/필터)
	- 상단 헤더(검색/프로필)
	- 메인 콘텐츠(카드 그리드) 구조 잡기
    
- **디자인 시스템:** Tailwind CSS를 활용한 감성적인 컬러 팔레트(Primary Green, Accent Colors) 정의.
    
- **데이터 연동:** `insights` 배열(Array)을 만들어 더미 데이터를 카드 UI로 렌더링.
### **Phase 2: 핵심 기능 구현 (Core Features)**

- **New Insight (글쓰기):** 모달창을 통해 제목, 카테고리, 날짜, 핵심 메시지를 입력받아 리스트 최상단에 추가.
    
- **Log 추가 기능:** 각 카드에 대해 '자기 투영(Reflect)', '실천 과제(Action)', '대화(Dialogue)', '토론 주제(Topic)' 등 하위 로그를 추가하는 모달 구현.
    
- **상태 관리:** 글의 상태(Ready -> Logged -> Internalized)에 따라 카드가 다른 섹션으로 이동하도록 로직 구현.

### **Phase 3: 사용자 경험 개선 (UX & Interaction)**

- **알림 시스템:** 리마인드 알림 및 시스템 공지 구현. `localStorage`를 활용해 '읽음' 상태 유지 (새로고침 해도 빨간 점 다시 안 뜸).
    
- **카테고리 확장:** 비문학, 문학, 신문기사, 영화, 공연, 미디어 등 6가지 카테고리로 세분화 및 필터링 기능 강화.
    
- **지식 지도 (Map):** 카테고리별 비중을 시각적인 막대그래프(Bar Chart)와 퍼센트로 표현 (데이터 연동).
### **Phase 4: 개인화 및 게이미피케이션 (Personalization)**

- **프로필 설정:** 이름 변경 기능 구현 (유효성 검사 포함).
    
- **프로필 사진 커스텀:**
    
    - **이니셜 모드:** 배경색 선택 시 이름의 첫 글자로 자동 생성 (DiceBear API).
        
    - **업로드 모드:** 로컬 이미지를 캔버스로 리사이징/압축하여 저장 (Base64).
        
- **레벨 시스템 (Gamification):** 활동량에 따라 Lv.0 ~ Lv.20까지 성장하는 경험치 바와 칭호 시스템 도입.

### **Phase 5: 시스템 고도화 (System Upgrade)**

- **다국어 지원 (i18n):** 한국어/영어 전환 기능 구현. (모든 텍스트를 `translations` 객체로 관리).
    
    - 날짜 포맷 자동 변환 (Sep 2025 ↔ 2025년 9월).
        
    - 카테고리 및 버튼 텍스트 실시간 번역.
        
- **다크 모드 (Dark Mode):** Tailwind `darkMode: 'class'`를 활용해 라이트/다크 테마 전환 기능 완벽 구현.
