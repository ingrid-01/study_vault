# 1. 포스트 수정 #

포스트들 - 해당 포스트를 클릭하면 edit/편집 가능하게 만들어야해. 그리고 잘못 만들었으면 지울 수 있는 버튼도 필요하고. 그리고 일단 포스트를 올리면 브라우저를 새로고침해도 유지될 수 있게. 지금은 예시로 넣은 포스트들이 다 하드 코딩되어있어. 이것도 바꿔

---

## 포스트 만들 때 ##

- 포스트 만들 떄 핵심 메시지 뿐 아니라 로그 할 때 더할 수 있는 자기 투영, 실천 과, 대화 로그, 토론 주제도 다 더할 수 있게 만들어. 이 네 가지는 사용자가 골라서 더 할 수 있게: + 버튼을 누르면 4중 하나를 고를 수 있고, 또 하나를 더하려면 밑에 붙여지는 방식으로 

- 포스트의 날짜 - 캘린더 뷰로 표기하기

- 포스트 옵션 - 창 토글 수정 + 한/영 설정 수정

- 현재 포스트들은 하드코드 되어있다. 새로운 통찰을 더하면 새로고침해도 유지 될 수 있게. + 현재 하드코딩 된 것은 없애기. 
	- 데이터 영속성 (Data Persistence)
			- 여태까지는 insights 배열 초기화되었음
			- 지금부터는 브라우저의 localStorage기능 활용, 데이터를 영구적으로 저장

- 포스트를 지우고 수정 할 수 있는 기능이 필요함.
	- 그리고 연필 아이콘이 날짜 위에 올라와 있어서 잘 안보여. - 줄 맞추기 정리
	- 기본 대시보드에 수정 버튼 (연필) 모양과 지우기 버튼이 눌러도 아무 반응이 없어. 그리고 포스트 카드를 누르고 들어가도 수정과 삭제가 되지 않아.
