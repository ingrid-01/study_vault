# 1. 개념 잡기 #
## 코어 JS 1장 - 얕은 복사 + 깊은 복사 원리 이해 ##

**자바스크립트가 데이터를 저장하는 방식 (메모리 구조)** 

- 내가 친구에게 필기 노트를 공유한 상황 비유
	1. 기본형 데이터 (숫자, 문자열 등) = 종이 복사
		- 친구가 내 노트를 복사기에 넣고 복사함
		- 친구가 복사본에 낙서? 원본 노트 괜춘, 노 영향
		
	2. 참조형 데이터 (객체, 배열 등) = 구글 드라이브 폴더 공유
		- 객체 사이즈 넘 큼 → 직접 복사 아니고 데이터가 있는 주소(링크)만 보냄
		- **그냥 할당 (=)**: 구글 드라이브 공유 링크 카톡으로 보냄
			- 친구가 그 링크로 들어와서 파일을 지우면? 내 파일도 지워짐 = 같은 곳을 바라보니까

**얕은 복사** & **깊은 복사**
- 얕은 복사 (Shallow Copy): 
	- 겉표지만 새로 만들고, 안에 있는 내용물(속성)은 여전히 링크로 연결된 상태
	- *비유*: 엑셀 파일의 '바로 가기 아이콘'만 바탕화면에 복사한 것과 비슷
		- 아이콘은 2개, 클릭해서 들어가면 결국 **같은 원본 파일**
		
- 깊은 복사 (Deep Copy): 
	- 안에 있는 내용물까지 완벽하게 새 종이에 필사해서 새로 만드는 것
	- *비유*: 엑셀 파일 자체를 복붙해서 사본파일을 만든 것. 
		- 사본을 수정해도 원본은 변하지 않음
---

### 🧐 멘토의 가이드 질문

이 비유를 바탕으로 '코어 자바스크립트' 1장의 핵심을 짚어봅시다.

만약 우리가 `{ 이름: "철수", 취미: ["축구", "게임"] }` 이라는 객체(사람) 정보를 복사한다고 가정해 볼게요.

**질문:** 이 객체를 **'얕은 복사'** 했을 때, 복사된 객체에서 `이름("철수")`을 바꾸는 것과 `취미(["축구", "게임"])` 목록을 수정하는 것이 서로 다른 결과를 초래할 수 있습니다. **왜 `취미`를 수정할 때만 원본에 영향을 미칠 가능성이 있을까요?** (힌트: `취미`는 배열이고, 배열도 객체라는 점을 생각해 보세요.)

답: 

> 왜냐하면 이름: ("철수")는 문자열이기 때문에 기본형 데이터야. 이는 복사를 해도 원본은 따로 있어서 복사본을 수정해도 원본에 영향을 미치지 않아. 하지만 취미: ["축구", "게임"]은 배열이기 때문에 참조형 데이터야. 이는 복사본과 원본이 같은 주소를 바라보고 있기 때문에 수정을 해버리면 원본과 복사본 둘 다 바뀌어버려.

- **이름("철수"):** 기본형 데이터라 값이 직접 복사되어서 서로 **독립적** 이다.
    
- **취미(배열):** 참조형 데이터라 메모리 **주소만 복사**되어서, 결국 같은 '방'을 공유하고 있는 셈이다.

**그렇다면!**

우리가 원하는 **'깊은 복사(Deep Copy)'**는 이 문제를 해결해야겠죠?

'얕은 복사'가 겉표지만 새로 만들고 내용물은 링크를 걸어둔 것이라면, **'깊은 복사'는 `취미` 같은 참조형 데이터를 만났을 때 어떻게 처리해야 원본과 완전히 끊어질 수 있을까요?**
