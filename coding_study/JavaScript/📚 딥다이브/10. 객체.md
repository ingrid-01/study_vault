### 📌 기술 면접 대비 핵심 요약 (JS Object)
반드시 알아야 할 4가지 핵심 기둥! 
- 면접관이 질문했을 때 바로 튀어나와야 하는 키워드들

**1. 객체 (Object)의 정의**
- 키워드: 키(Key), 값(Value)의 집합, 프로퍼티(Property), 메소드(Method)
- 핵심: JS는 객체 기반의 스크립트 언어이며, 원시 타입(Primitive)을 제외한 거의 모든 것(함수, 배열 등)이 객체이다. 값이 함수일 경우 "Method"라고 부른다. 

**2. 객체 생성 방법 (3가지)**
- 객체 리터럴: `{...}` 가장 일반적이고 간편한 방법
- Object 생성자 함수: `new Object()` 빈 객체 생성 후 프로퍼티 추가
- 생성자 함수 `new Person(...)` 템프릿처럼 여러 개이 동일한 구조의 객체를 생성할 때 사용. (this 바인딩 중요!)

**3. 프로퍼티 접근 및 조작**
- 접근: 마침표 표기법 (.)과 대괄호 표기법 (`['key'])`. (키가 유효한 식별자가 아니거나 변수일 때는 대괄호 필수!)
- 순회: `for-in` 문 (단, 순서가 보장되지 않고 배열에는 비추천)
- 동적 생성/삭제: 실행 중에 프로퍼티를 추가하거나 `delete` 연산자로 삭제 가능

**4. Pass-by-reference (참조에 의한 전달)** ⭐️⭐️⭐️
- 핵심: 객체는 값이 복사되는 것이 아니라 참조 값(메모리 주소)가 전달된다. 따라서 변수를 복사한 뒤 내용을 수정하면 원본도 함께 바뀐다.
