- 자바스크립트는 동적 타입 (dynamic type) 언어
	- 변수에 값 할당 =/= 예측

```js
function sum(a, b) {
	return a + b;
}
```
위 코드 의도:
- 2개의 number 타입 인수를 전달받음
- 그 합계를 반환하려는 것

BUT 코드 상:
- 어떤 타입의 인수를 전달하여야 하는지 + 어떤 타입의 값을 반환해야 하는지
- 명확하지 않다

Result:
```js
function sum(a, b) {
	 return a + b;
}

sum('x', 'y'); // 'xy'
```
- 자바스크립트 엔진은 아무런 이의 제기없이 위 코드 실행
	- 변수, 반환값의 타입을 사전에 지정하지 않는 자바스크립트의 동적 타이핑에 의한 것
	- 타입체크가 필요!

## 1. typeof

- Type Operator(타입 연산자)는 `typeof` → 피연산자의 데이터 타입을 문자열로 반환
  
```js
typeof ''; 
typeof 1;
typeof NaN;
typeof true;
```

- null → object
- Date, RegExp, 사용자 정의 객체 등 → object
- 함수 → function
- typeof = null을 제외한 원시 타입 체크엔 문제 X 
	- 객체 종류까지 구분하여 체크하긴 힘듦
		- 여러 종류의 객체 (일반, 배열, Date, RegExp, Function, DOM 요소 등)을 구분할 수 있는 타입 체크 기능을 만드려면!!!

## 2. Object.prototype.toString

- Object.prototype.toString 메소드 → 객체를 나타내는 문자열 반환하기
```js
var obj = new Object();
obj.toString(); // [object Object]
```

- Function.prototype.call 메소드 → 모든 타입의 값의 타입을 알아낼 수 있다
```js
Object.prototype.toString.call('');  // [object String]
Object.prototype.toString.call(new String());  // [object String]

```

- 타입을 반환하는 함수
```js
function getType(target) {
	return Object.prototype.toString.call(target).slice(8, -1);
}
```

- String.prototype.slice → Object.prototype.toString.call에서 `[object]` 빼고 타입만 추출
```js
getType(''); // String
getType(1); // Number
```
